前端响应式设计

发表于 2017-08-01 | 分类于 前端

响应式页面设计（Responsive Web Design）简单来说就是同一个页面可以适应不同屏幕大小设备的设计方案

viewport

在手机上做页面比较重要的就是设置viewport ，通俗的讲，移动设备上的viewport就是设备的屏幕上能用来显示我们的网页的那一块区域，如果不设置viewport浏览器就会默认渲染为980像素的pc页面。

很多人常常使用initial-scale=1到非响应式网站上，这会让网站以100%宽度渲染而不会自动缩放，用户需要手动移动页面或者缩放。最差的是和initial-scale=1同时使用user-scalable=no或maximum-scale=1，这将使你的网站不能被缩放——用户不能放大/缩小网页来看到全部的内容。所以，请记住：如果你的网站不是响应式的，请不要使用initial-scale或者禁用缩放。

响应式的图片

max-width：100%

当给图片设置max-width:100%时，大图就会随容器自动缩放，保持高宽比。 当容器比较大时图片会按照原本尺寸显示，容器比较小时就会填满容器不会出现滚动条

背景图片

background-size：cover：图片会覆盖整个容器，但是可能会造成背景图片被裁切，适用于背景图片不重要时。
background-size：contain：容器完整显示出背景图片，不会造成背景图片被裁切，适用于背景图片比较重要时。

可以查询的media

  • width浏览器宽度
  • height浏览器高度
  • device-width设备屏幕分辨率的宽度值
  • device-height设备屏幕分辨率的高度值
  • orientation浏览器窗口的方向纵向还是横向，当窗口的高度值大于等于宽度时该特性值为portrait，否则为landscape
  • aspect-ratio比例值，浏览器的纵横比
  • device-aspect-ratio比例值，屏幕的纵横比
